# Global Reporting Recipe / Constitution

## Description
A master agent recipe that defines Block's reporting requirements, standards, and conventions. All reporting sub-agents inherit from this recipe before applying their specific deliverable logic.

## Status
Active

## Purpose
- Establish a single source of truth for how reports are structured, populated, and communicated
- Ensure consistency across all automated reporting outputs
- Serve as the foundation layer for sub-agents (weekly report, flash update, etc.)

## Key Dates / Deadlines
None currently — ongoing buildout

## Next Steps
- Define the global recipe structure
- Document reporting requirements and standards
- Test against a real report before sub-agents are built on top
