# Block Pacing Dashboard

Live 2-page business performance dashboard tracking current quarter pacing vs. consensus and forward-looking quarterly forecasts. Deployed to Block Cell for executive access.

**Production URL:** https://blockcell.sqprod.co/sites/nmart-pacing-dashboard/

## Overview

The dashboard complements the weekly Block Performance Digest (published Tuesdays) by providing a continuously-updated view of the same metrics. Leadership can bookmark the Block Cell URL and check pacing at any time without waiting for the next Tuesday publish.

### Page 1: Q1 Pacing

Current quarter view with live data from the master pacing sheet.

- **Key Takeaways** — two-bullet executive summary (GP + AOI headlines)
- **Block Scorecard** — 3 KPI cards: Block Gross Profit, Adjusted Operating Income, Rule of 40
- **Block Consolidated Table** — GP, AOI, AOI Margin, Rule of 40, GP Net of Risk Loss with Q1 Pacing, WoW delta, YoY Growth, AP, Guidance, Consensus, Consensus YoY, and vs. Consensus columns
- **Brand Scorecard** — 2 KPI cards: Cash App Actives (green), Square Global GPV (blue)
- **Cash App Table** — GP, Actives, Inflows per Active, Monetization Rate vs. consensus
- **Square Table** — GP, Global/US/International GPV vs. consensus

### Page 2: FY2026 Outlook

Multi-quarter forward view combining internal forecast with Street consensus.

- **Key Takeaways** — auto-generated analysis highlighting GP trajectory and AOI margin divergence
- **Block Gross Profit** — Q1-Q4 internal forecast vs. consensus, YoY growth, dollar and growth deltas
- **Adjusted Operating Income** — Q1-Q4 with margin comparison (internal margin vs. consensus margin, margin delta in pts)
- **GP Net of Risk Loss** — Q1-Q4 internal vs. consensus with YoY growth comparison

## Files

| File | Purpose |
|------|---------|
| `dashboard.html` | The 2-page HTML dashboard. Self-contained CSS + JS. Loads data from `dashboard_data.js`. |
| `dashboard_data.js` | Generated data file (JSON wrapped in a JS variable). **Not checked in** — contains live financial data. Generated by `refresh.py`. |
| `refresh.py` | Python extraction script. Reads three cached JSON files from `/tmp/`, extracts and formats all metrics, writes `dashboard_data.js`. |
| `refresh.sh` | Shell wrapper. Authenticates, reads all three Google Sheets via `gdrive-cli`, then runs `refresh.py`. One command for a full refresh. |

## Refresh Workflow

### Data refresh (numbers change in sheets)

```bash
cd ~/Desktop/Nick's\ Cursor/Pacing\ Dashboard/
./refresh.sh
```

This takes ~30 seconds:
1. Checks Google auth via `gdrive-cli`
2. Reads all three sheets → caches as JSON in `/tmp/`
3. Runs `refresh.py` → generates `dashboard_data.js`
4. Prints a summary of key metrics for verification

## Formatting Standards

Follows the same formatting rules as the weekly Block Performance Digest:

- **Dollars:** $B with 2 decimals ($2.90B), $M with 0 decimals for ≥$10M ($638M), 1 decimal for <$10M ($4.2M)
- **Percentages:** No decimal for ≥10% (+27%), 1 decimal for <10% (+9.0%). GPV YoY in 10-15% range keeps 1 decimal (+13.1%).
- **Basis points:** Integer with space (+34 bps)
- **Actives:** Always 1 decimal (58.5M)
- **Negatives:** Parenthetical — ($5M), (1.6%), (5 bps)
- **Deltas:** Always include +/- sign
- **Growth deltas:** Formatted in points (+5 pts, (8 pts))
- **Signal colors:** Green (>+0.5% vs comparison), Red (<-0.5%), Neutral (±0.5%)
